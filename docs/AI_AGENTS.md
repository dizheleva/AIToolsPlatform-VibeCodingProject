# AI Agents Documentation

## Общ преглед

Тази платформа поддържа работа с AI агенти за автоматизация на задачи и подобряване на производителността. AI агентите могат да се използват за различни цели като разработка, тестване, документация и управление на инструменти.

## Типове AI агенти

### 1. Development Agent (Агент за разработка)

**Предназначение:** Помощ при разработка на код, рефакториране, отстраняване на грешки и подобряване на качеството на кода.

**Основни функции:**
- Анализ на код и предложения за подобрения
- Генериране на тестове
- Рефакториране на съществуващ код
- Отстраняване на грешки и проблеми
- Генериране на документация за код

**Използване:**
Агентът за разработка може да се стартира с начални промтове (виж [Development Prompts](./DEVELOPMENT_PROMPTS.md)) за конкретни задачи.

### 2. Code Review Agent (Агент за преглед на код)

**Предназначение:** Автоматичен преглед на код за качество, сигурност и съответствие с best practices.

**Основни функции:**
- Проверка за security vulnerabilities
- Проверка за code smells
- Валидация на coding standards
- Предложения за оптимизация

### 3. Documentation Agent (Агент за документация)

**Предназначение:** Автоматично генериране и актуализация на документация.

**Основни функции:**
- Генериране на API документация
- Актуализация на README файлове
- Създаване на техническа документация
- Генериране на коментари в код

### 4. Testing Agent (Агент за тестване)

**Предназначение:** Автоматично генериране и изпълнение на тестове.

**Основни функции:**
- Генериране на unit тестове
- Генериране на integration тестове
- Проверка на code coverage
- Изпълнение на тестове и репортиране

## Интеграция с платформата

### API Endpoints за AI агенти

В бъдеще ще бъдат добавени специални endpoints за работа с AI агенти:

```
POST /api/agents/development/start
POST /api/agents/development/execute
GET  /api/agents/development/status
POST /api/agents/code-review/analyze
POST /api/agents/documentation/generate
```

### Конфигурация

AI агентите могат да се конфигурират чрез environment variables:

```env
AI_AGENT_PROVIDER=openai|anthropic|local
AI_AGENT_API_KEY=your_api_key
AI_AGENT_MODEL=gpt-4|claude-3|local-model
AI_AGENT_TEMPERATURE=0.7
AI_AGENT_MAX_TOKENS=2000
```

## Best Practices

### 1. Използвай конкретни промтове

Винаги използвай конкретни и ясни промтове. Избягвай неясни инструкции.

**Лошо:**
```
"Поправи кода"
```

**Добре:**
```
"Рефакторирай метода calculateTotal() в OrderController за да използва dependency injection вместо директна инстанциация на OrderService"
```

### 2. Предоставяй контекст

Винаги предоставяй достатъчно контекст за задачата - файлове, структура на проекта, изисквания.

### 3. Итеративен подход

Работи с агента итеративно - започни с малка задача, провери резултата, след това разшири.

### 4. Проверявай резултатите

Винаги проверявай и тествай резултатите от агента преди да ги комитираш.

### 5. Използвай version control

Винаги използвай git за да можеш да се върнеш назад ако нещо не работи.

## Безопасност

### Важни предупреждения

1. **Никога не предоставяй чувствителни данни** (API ключове, пароли, лични данни) в промтовете
2. **Проверявай генерирания код** за security vulnerabilities
3. **Не използвай агент за production код** без ревю
4. **Ограничавай достъпа** до AI агент функционалността само за одобрени потребители

### Рекомендации

- Използвай AI агенти само в development среда
- Винаги прави code review на генерирания код
- Тествай всички промени преди deployment
- Документирай използването на AI агенти в commit messages

## Примери за използване

### Пример 1: Рефакториране на код

```bash
# Стартирай Development Agent с промт за рефакториране
POST /api/agents/development/start
{
  "prompt": "Рефакторирай UserController::index() метода за да използва repository pattern вместо директни Eloquent заявки",
  "files": ["app/Http/Controllers/UserController.php"],
  "context": "Laravel 12, PHP 8.2"
}
```

### Пример 2: Генериране на тестове

```bash
# Генерирай тестове за контролер
POST /api/agents/testing/generate
{
  "target": "app/Http/Controllers/AiToolController.php",
  "type": "feature",
  "coverage": ["index", "store", "show", "update", "destroy"]
}
```

### Пример 3: Генериране на документация

```bash
# Генерирай API документация
POST /api/agents/documentation/generate
{
  "type": "api",
  "routes": ["/api/tools", "/api/categories"],
  "format": "markdown"
}
```
